# コーディングおよびドキュメントスタイル規約

## ドキュメント規則 (`.cursorrules`)
ドキュメントを生成・変更する場合は、以下の規則に厳密に従う必要があります。
1. **英語と日本語の2ヶ国語対応 (Dual-Language Documentation)**:
   - すべての公開ドキュメントは英語をプライマリバージョンとします。
   - 英語バージョンと同時に、必ず対応する日本語バージョンを作成します。
   - 両ファイルの上部には、相互へのリンクを必ず配置します。
     - 英語版上部: `[日本語版 (Japanese)](./filename_ja.md)`
     - 日本語版上部: `[English Edition](./filename.md)`
2. **ルート README**:
   - ルートディレクトリにあるエントリーポイントファイル：
     - `/README.md` (英語版)
     - `/README_ja.md` (日本語版)
3. **公開ドキュメントディレクトリ**:
   - その他の公開ドキュメントファイルは `/docs` ディレクトリ以下に保存します。
     - 例: `/docs/architecture.md` (英語), `/docs/architecture_ja.md` (日本語)
4. **ローカル作業およびAI思考プロセス領域**:
   - `/docs/local` ディレクトリはローカル作業、個人のメモ、AI思考プロセス用領域として予約されています。
   - このディレクトリは `.gitignore` に含まれており、リポジトリにコミットしてはいけません。ここにあるファイルは翻訳不要です。

## コーディング規約
- **バックエンド (Go)**
  - 標準的な Go フォーマットルール (`go fmt`) を適用します。
  - 変数名や関数名は、プロジェクトの既存コードに合わせ、わかりやすい英語名（キャメルケース）を使用します。
  - ドキュメンテーションコメントやコード内コメントはすべて**日本語**で記述します。
- **フロントエンド (TypeScript / Svelte)**
  - Svelte 5 のルーン (Runes) (`$state`, `$effect`, `$derived` など) を使用してリアクティブ状態を管理します。
  - スタイル調整には Tailwind CSS v4 の構成に従います。
  - 変数名、関数名は英語（キャメルケース）で命名し、ローマ字命名は禁止します。
  - コメント（docstring含む）はすべて**日本語**で記述します。
