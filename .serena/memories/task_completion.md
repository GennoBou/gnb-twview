# タスク完了時のチェックリスト

開発タスクが完了し、変更をマージまたはコミットする前に、以下の項目を確認・実行してください。

## 1. コードの検証
- **自動フォーマットの適用**:
  - Go バックエンド: `go fmt ./...` を実行し、すべてのファイルが整形されていること。
  - フロントエンド: エディタの自動フォーマットなどを適用すること。
- **静的解析・型チェック**:
  - フロントエンドの TypeScript/Svelte 型チェックを実行する:
    ```powershell
    cd frontend; npm run check
    ```
    エラーが発生しないことを確認します。
- **テストの実行**:
  - Go バックエンドのテストを実行する:
    ```powershell
    go test ./...
    ```
    すべてのテストがパスすることを確認します。

## 2. ドキュメントの同期 (`.cursorrules` 準拠)
- タスク中に新しい機能や設計の変更に伴い公開ドキュメントを作成・更新した場合、英語および日本語の2ヶ国語バージョンが両方作成されていることを確認します。
- 各ファイル（英語版と日本語版）の最上部に、もう一方のバージョンへのクロスリンクが記載されているか確認します。
  - 英語版: `[日本語版 (Japanese)](./filename_ja.md)`
  - 日本語版: `[English Edition](./filename.md)`
- ローカル作業用、AI用のメモは `/docs/local/` の下にのみ置かれ、Gitの対象外になっていることを確認します。

## 3. コミット・PR前チェック
- 余分な一時ファイルやデバッグ用のコード（不要な `fmt.Println` や `console.log` など）が残っていないか確認します。
- コード内のコメントや docstring はすべて**日本語**で記述されているか確認します。
- コマンド操作（特にファイルの削除や移動などの破壊的操作）を伴う手順書やスクリプトがある場合、PowerShell向けに正規のコマンドレット (`Remove-Item`, `Get-ChildItem` など) が使用されていることを確認します。
