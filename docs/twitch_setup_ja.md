# Twitch Client ID 取得・設定手順書

本アプリケーション（`gnb-twview`）で Twitch API を利用し、フォローしている配信者のライブ情報を取得するために必要な、**Client ID（クライアントID）** の取得方法と設定方法を解説します。

---

## 1. Twitch 開発者ポータルでのアカウント登録

まずは Twitch の開発者ポータルにログインし、開発者としてのアカウント登録（必要な場合は2要素認証の設定）を行います。

1. **[Twitch Developer Portal](https://dev.twitch.tv/)** にアクセスします。
2. 右上の **[Log In]** から、普段配信視聴に使っている Twitch アカウントでログインします。
3. ログイン後、同じく右上の **[Developer Console]** ボタンをクリックします。
4. （初回の場合）開発者としての規約への同意や、アカウントへの2段階認証（2FA）の設定が求められた場合は、画面の指示に従って有効化してください。

---

## 2. アプリケーションの登録

API を利用するために、あなたの専用アプリケーションを Twitch に登録します。

1. Developer Console の左メニューから **[Applications]** を選択します。
2. 画面右上にある **[+ Register Your Application]** ボタンをクリックします。
3. 以下の項目を入力・設定します。

| 項目名 | 設定内容 | 説明 |
| :--- | :--- | :--- |
| **Name** | `gnb-twview` （任意） | 他のアプリと重複しない一意の名前を入力してください。 |
| **OAuth Redirect URLs** | `http://localhost:8520/callback` | **非常に重要です。** 本アプリが認証情報（認可コード）を受け取るためのローカルアドレスです。必ず正確に入力し、入力後に **[Add]** ボタンを押して追加してください。 |
| **Category** | `Application Integration` または `Other` | アプリケーションのカテゴリを選択します。 |
| **Client Type** | `Public` | 本アプリはデスクトップアプリケーション（シークレット情報を安全に隠せないクライアント）であるため、**Public** を選択します。 |

4. ロボットではないことの確認（reCAPTCHA）にチェックを入れ、**[Create]** ボタンをクリックします。

---

## 3. Client ID の確認

1. 登録が完了すると、アプリケーション一覧に登録したアプリが表示されます。
2. 対象アプリの **[Manage]** ボタンをクリックします。
3. 画面に表示されている **Client ID** （英数字のランダムな文字列）をコピーします。
   > [!NOTE]
   > **Client Secret（クライアントシークレット）について**
   > 画面に「Client Secret」の生成ボタン（New Secret）がありますが、本アプリではセキュリティ上の安全対策（PKCE）を導入しているため、**Client Secret は使用しません。生成・コピーする必要はありません。**

---

## 4. 適切な権限（スコープ）について

本アプリは、あなたがフォローしている配信者の情報を取得するために、以下の権限（スコープ）を要求します。

* **`user:read:follows`**
  * **説明**: ログインしたユーザーがフォローしているチャンネル（配信者）のリストを読み取るための権限です。
  * **用途**: フォローしているチャンネルが現在ライブ配信中かどうかを Twitch API（`/helix/streams/followed`）で確認するために必要となります。この権限がないと、ライブ中の配信者リストを表示できません。

---

## 5. アプリケーション（コード）への適用

取得した Client ID を本アプリに適用します。

1. 本プロジェクトの [backend/twitchservice.go](file:///d:/Data/Projects/GitHub/gnb-twview/backend/twitchservice.go) を開きます。
2. 23行目付近にある `TwitchClientID` 定数の値を、コピーした新しい Client ID に書き換えます。

```go
// backend/twitchservice.go 23行目付近
const TwitchClientID = "あなたの新しいClient IDをここに貼り付けます"
```

3. アプリケーションをビルド・起動し、正常にログインできるか確認します。
